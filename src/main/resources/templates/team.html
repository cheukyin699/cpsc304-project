Project Introduction

The domain we are going to model is genetic disorders. We want to design a database to keep track of the related information on genetic disorders for physicians to use for diagnostic purposes. The aspects of genetic diseases that we will be modeling will be about the diagnostic features and methods of those diseases. This includes the type of diseases, medical diagnostic tests, inheritance pattern, symptoms, and markers for diagnosis and prognosis. There will be a number of different diseases and subtypes of each disease. Some of them may share the same diagnostic tests, but depending on the disease, different biological markers are associated.

We anticipate two different types of users - physicians who will be able to use the database as a diagnostic tool for genetic disorders and researchers who will be able to insert new disorders or modify attributes of existing disorders. Physicians will be able to query presenting signs and symptoms to determine which diagnostic tests to perform, and then query the test results to receive a list of possible diagnoses. Researchers will be able to modify the information in the database to update details of diseases as new diagnostic markers, treatment options, and subtypes are identified.
